RigLeConnectionManagerDelegate Protocol Reference

Conforms to NSObject
Declared in RigLeConnectionManager.h

Overview

This protocol provides events to the object that requested a connection or disconntion.

– didConnectDevice: required method

This method is called when a connection is successfully made to a CBPeripheral. At this point however, no discovery of services or characteristics has been performed. This is acheieved by calling runDiscovery on the object.

- (void)didConnectDevice:(RigLeBaseDevice *)device

Parameters

device

The RigLeBaseDevice object created after a successful connection.

@seealso RigLeBaseDevice

– deviceConnectionDidFail: required method

This method is called when a failure occurs and CoreBluetooth was unable to successfully connect to the device. Note that this method will only be called if the central manager reports the connection failed.

- (void)deviceConnectionDidFail:(RigAvailableDeviceData *)device

Parameters

device

The device data for which the connection failed.

– deviceConnectionDidTimeout: required method

This method is called when the timeout value passed to connectDevice:connectionTimeout: elapses prior to a successful connection being made.

- (void)deviceConnectionDidTimeout:(RigAvailableDeviceData *)device
