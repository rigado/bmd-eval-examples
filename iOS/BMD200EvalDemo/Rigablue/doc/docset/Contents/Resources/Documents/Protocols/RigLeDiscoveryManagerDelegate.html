RigLeDiscoveryManagerDelegate Protocol Reference

Conforms to NSObject
Declared in RigLeDiscoveryManager.h

Overview

@protocol RigLeDiscoveryManagerDelegate

This protocol provides an interface for the RigLeDiscoveryManager to send event information back to the object that requested the discovery operation.

– didDiscoverDevice: required method

@method disDiscoverDevice:

- (void)didDiscoverDevice:(RigAvailableDeviceData *)device

Parameters

device

A RigAvailableDeviceData object.

Discussion

This method is called at any time a device is dicovered assuming the device has not already been discovered during the current discovery session. After discovery, applications will use the device objects to initial connections to discovered devices.

– discoveryDidTimeout required method

@method discoveryDidTimeout

- (void)discoveryDidTimeout

Discussion

If a timeout value is provided when a discovery session is initiated, this method is called when that timeout has elapsed. Once this method is called, discovery is no longer being performed an new devices will not continue to be discovered until a new discovery session is started.
